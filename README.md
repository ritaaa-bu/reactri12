# 📱 ProductivityApp - Мобильное приложение для повышения продуктивности

Приложение для управления повседневными активностями и повышения личной продуктивности. Разработано на React Native CLI (без Expo) с использованием современных практик и паттернов.

## 🎯 Описание проекта

Приложение позволяет пользователям:
- Регистрироваться и создавать личный аккаунт
- Добавлять ежедневные активности (задачи)
- Отмечать выполнение задач
- Просматривать статистику выполненных дел
- Переключаться между светлой и тёмной темой
- Фильтровать задачи по статусу

## 🚀 Как запустить проект

### Предварительные требования

1. **Node.js** (версия 18 или выше)
   - Скачать: https://nodejs.org/
   - Проверка: `node --version`

2. **npm** (идёт в комплекте с Node.js)
   - Проверка: `npm --version`

3. **Для Android:**
   - Android Studio: https://developer.android.com/studio
   - Android SDK (устанавливается через Android Studio)
   - Эмулятор или реальное устройство
   - Переменные окружения:
     ```bash
     # Windows (добавить в системные переменные)
     ANDROID_HOME=C:\Users\<User>\AppData\Local\Android\Sdk
     
     # macOS/Linux (~/.bashrc или ~/.zshrc)
     export ANDROID_HOME=$HOME/Android/Sdk
     export PATH=$PATH:$ANDROID_HOME/emulator
     export PATH=$PATH:$ANDROID_HOME/platform-tools
     ```

4. **Для iOS (только macOS):**
   - Xcode из App Store
   - CocoaPods: `sudo gem install cocoapods`

### Пошаговая установка

#### Шаг 1: Клонирование/создание проекта
```bash
cd /path/to/project
```

#### Шаг 2: Установка зависимостей
```bash
npm install
```

#### Шаг 3: Запуск метро-сервера (упаковщика)
```bash
npm start
```

#### Шаг 4: Запуск приложения

**Для Android:**
```bash
# Запуск на эмуляторе
npm run android

# Или напрямую
npx react-native run-android

# На конкретном устройстве (получить ID через adb devices)
npx react-native run-android --deviceId=<device_id>
```

**Для iOS (macOS):**
```bash
# Установка pod-зависимостей (только для iOS)
cd ios && pod install && cd ..

# Запуск на симуляторе
npm run ios

# Или напрямую
npx react-native run-ios

# На конкретном симуляторе
npx react-native run-ios --simulator="iPhone 15"
```

## 🔄 Поток данных

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────┐
│   AsyncStorage  │ ←→  │  storage.js      │ ←→  │   Screens   │
│  (локальное     │     │  (функции CRUD)  │     │             │
│   хранилище)    │     │                  │     │             │
└─────────────────┘     └──────────────────┘     └─────────────┘
                                ↑
                                │
                        ┌───────────────┐
                        │ themeContext  │
                        │ (глобальное   │
                        │  состояние)   │
                        └───────────────┘
                                ↑
                                │
                        ┌───────────────┐
                        │  Components   │
                        │  (UI элементы)│
                        └───────────────┘
```

### Как это работает:

1. **Данные пользователя** сохраняются в AsyncStorage через `storage.js`
2. **Тема приложения** хранится в AsyncStorage и управляется через React Context
3. **Экраны** вызывают функции из `storage.js` для чтения/записи данных
4. **Компоненты** получают тему через хук `useTheme()` из контекста
5. **Навигация** управляет переходами между экранами

## 📚 Как объяснить на защите диплома

### Ключевые моменты для презентации:

1. **Выбор технологий:**
   - React Native CLI выбран для полного контроля над нативными модулями
   - JavaScript вместо TypeScript для упрощения кода и быстрого прототипирования
   - React Navigation v6 — стандарт де-факто для навигации в RN
   - AsyncStorage — простое локальное хранилище без необходимости бэкенда

2. **Архитектурные решения:**
   - Модульная структура: каждый компонент в отдельном файле
   - Разделение ответственности: screens, components, utils, theme
   - React Hooks (useState, useEffect, useContext) вместо классовых компонентов
   - Context API для глобального состояния (тема) вместо Redux

3. **Особенности реализации:**
   - Две темы с сохранением выбора пользователя
   - Валидация форм на клиенте
   - Фильтрация и сортировка данных
   - Адаптивный дизайн под разные размеры экранов

4. **Расширяемость:**
   - Легко добавить новые экраны
   - Можно подключить бэкенд (заменить AsyncStorage на API вызовы)
   - Возможность добавить push-уведомления, синхронизацию

### Возможные вопросы и ответы:

**Q: Почему не использовали Redux?**
> A: Для данного приложения достаточно Context API. Redux добавил бы избыточную сложность без существенных преимуществ при текущем масштабе.

**Q: Как обеспечить безопасность паролей?**
> A: В текущей версии пароли не сохраняются. В production-версии нужно использовать хеширование (bcrypt) и отправлять данные на сервер по HTTPS.

**Q: Почему не Expo?**
> A: Expo ограничивает доступ к нативным модулям. CLI даёт полный контроль и гибкость для будущей кастомизации.

## 🛠️ Решение распространённых проблем

### Ошибка: "Command run-android not recognized"
```bash
# Очистить кэш и переустановить
rm -rf node_modules
npm install
cd android && ./gradlew clean && cd ..
```

### Ошибка линковки библиотек (iOS)
```bash
cd ios
pod deintegrate
pod install
```

### Метро-сервер не запускается
```bash
# Очистить кэш метро
npx react-native start --reset-cache
```

### Ошибки сборки Android
```bash
# Очистить сборку
cd android
./gradlew clean
./gradlew build
```

### Проблемы с эмулятором Android
- Убедитесь, что эмулятор запущен
- Проверьте: `adb devices`
- Перезапустите adb: `adb kill-server && adb start-server`
