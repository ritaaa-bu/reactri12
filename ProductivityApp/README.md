# ProductivityApp - Инструмент повышения личной продуктивности

Мобильное приложение для управления повседневными активностями, разработанное на React Native CLI.

## 📁 Структура проекта

```
ProductivityApp/
├── App.js                          # Главный файл приложения (навигация)
├── package.json                    # Зависимости и конфигурация
├── index.js                        # Точка входа React Native
├── app.json                        # Конфигурация приложения
├── babel.config.js                 # Конфигурация Babel
├── metro.config.js                 # Конфигурация Metro bundler
└── src/
    ├── screens/
    │   ├── HomeScreen.js           # Главная страница / Дашборд
    │   ├── AddActivityScreen.js    # Экран добавления активности
    │   ├── HistoryScreen.js        # Экран истории и статистики
    │   └── SettingsScreen.js       # Экран настроек
    ├── components/                 # Переиспользуемые компоненты (опционально)
    ├── navigation/                 # Конфигурация навигации (опционально)
    └── utils/                      # Вспомогательные функции (опционально)
```

## 🚀 Инструкция по запуску

### Предварительные требования

1. **Node.js** версии 18 или выше
   - Проверка: `node --version`
   - Скачать: https://nodejs.org/

2. **React Native CLI**
   - Устанавливается автоматически при создании проекта

3. **Для Android:**
   - Android Studio с JDK 17
   - Android SDK (API level 29+)
   - Эмулятор или физическое устройство
   - Переменная окружения `ANDROID_HOME`

4. **Для iOS (только macOS):**
   - Xcode 15+
   - CocoaPods (`sudo gem install cocoapods`)

### Шаг 1: Клонирование/создание проекта

```bash
cd /workspace/ProductivityApp
```

### Шаг 2: Установка зависимостей

```bash
npm install
```

### Шаг 3: Запуск Metro Bundler

В отдельном терминале:
```bash
npm start
```

Или:
```bash
npx react-native start
```

### Шаг 4: Запуск на эмуляторе/устройстве

#### Для Android:

```bash
# Запуск на подключенном устройстве или эмуляторе
npm run android

# Или явно:
npx react-native run-android

# Если несколько устройств, укажите ID:
npx react-native run-android --deviceId=<device_id>
```

#### Для iOS (macOS):

```bash
# Установка pod-зависимостей
cd ios && pod install && cd ..

# Запуск на симуляторе
npm run ios

# Или явно:
npx react-native run-ios
```

### Шаг 5: Отладка

- **Android:** Нажмите `Ctrl+M` (эмулятор) или встряхните устройство
- **iOS:** Нажмите `Cmd+D` (симулятор) или встряхните устройство
- Используйте Chrome DevTools: `chrome://inspect`

## 🛠 Решение常见问题

### Ошибка "SDK location not found" (Android)
Создайте файл `android/local.properties`:
```
sdk.dir=/path/to/your/Android/Sdk
```

### Ошибка "Command failed: ./gradlew" (Android)
```bash
cd android
./gradlew clean
cd ..
npm run android
```

### Ошибка с кэшем Metro
```bash
npm start -- --reset-cache
```

### Ошибка pod install (iOS)
```bash
cd ios
pod deintegrate
pod install
cd ..
```

## 📝 Описание экранов

### 1. HomeScreen (Главная)
- Приветственный блок
- Карточка прогресса с визуализацией
- Список сегодняшних активностей
- Плавающие кнопки навигации
- Функция отметки выполнения (toggle)

### 2. AddActivityScreen (Добавить активность)
- Форма с валидацией полей
- Выбор категории через чипы
- Поля: название, категория, время, заметки
- Кнопки сохранения и отмены

### 3. HistoryScreen (История)
- Карточки статистики (всего, выполнено, эффективность)
- Переключатель периода (день/неделя/месяц)
- Список истории с индикаторами статуса

### 4. SettingsScreen (Настройки)
- Переключатели: уведомления, тема, звук, вибрация
- Выбор языка
- Управление данными (экспорт/импорт/сброс)
- Информация о приложении

## 🏗 Архитектурные решения

### 1. Одноуровневая структура экранов
Все экраны находятся в папке `src/screens/`, что упрощает навигацию и поддержку.

### 2. Стек-навигация
Используется `native-stack` для производительности и нативных переходов.

### 3. Локальное состояние (useState)
Для учебного проекта не требуется глобальный state-менеджер. Каждый экран управляет своим состоянием.

### 4. useEffect для жизненного цикла
Загрузка данных происходит при монтировании компонента.

### 5. StyleSheet для стилей
Все стили вынесены в отдельные объекты для переиспользования и читаемости.

### 6. Модульность
Каждый экран — независимый компонент, что позволяет легко тестировать и модифицировать.

## 📚 Объяснение для дипломной работы

### Ключевые моменты для защиты:

1. **React Native CLI vs Expo**
   - Выбран CLI для полного контроля над нативными модулями
   - Возможность кастомизации под конкретные требования

2. **Функциональные компоненты с хуками**
   - Современный подход React (после 16.8)
   - useState для локального состояния
   - useEffect для побочных эффектов

3. **Навигация**
   - React Navigation v6 — стандарт индустрии
   - Stack Navigator для иерархических переходов

4. **UI/UX решения**
   - Glassmorphism для современного вида
   - Адаптивная верстка через Flexbox
   - Обратная связь через тактильные элементы

5. **Расширяемость**
   - Легко добавить новые экраны
   - Можно интегрировать AsyncStorage для сохранения данных
   - Возможно подключение API для синхронизации
