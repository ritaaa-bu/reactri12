# 📚 Пояснительная записка для дипломной работы

## Тема: «Разработка инструмента повышения личной продуктивности на основе повседневных активностей»

---

## 1. Введение

### 1.1 Актуальность темы
В современном мире управление временем и личная продуктивность становятся критически важными навыками. Мобильные приложения для трекинга активностей помогают пользователям структурировать свой день, отслеживать прогресс и формировать полезные привычки.

### 1.2 Цель работы
Разработка кроссплатформенного мобильного приложения на React Native CLI для управления повседневными активностями с современным интерфейсом в стиле glassmorphism.

### 1.3 Задачи
- Проектирование архитектуры приложения
- Реализация навигации между экранами
- Создание UI-компонентов с эффектом стекла
- Разработка функционала добавления и отслеживания активностей
- Реализация статистики и истории

## 2. Технологический стек

### 2.1 React Native CLI
**Почему не Expo?**
- Полный контроль над нативными модулями
- Возможность кастомизации под специфические требования
- Прямой доступ к Android/iOS API
- Меньше ограничений при публикации в сторах

### 2.2 JavaScript ES6+
- Функциональные компоненты вместо классов
- Хуки useState и useEffect для управления состоянием
- Стрелочные функции для лаконичности кода
- Деструктуризация и spread-операторы

### 2.3 React Navigation v6
- Stack Navigator для иерархической навигации
- Нативные анимации переходов
- Типизированная навигация (в JS — через PropTypes)
- Глубокая ссылка (deep linking) поддержка

### 2.4 StyleSheet
- Встроенное решение React Native
- Оптимизация рендеринга (стили создаются один раз)
- Платформенно-специфичные стили
- Поддержка темизации

```
        ├── HomeScreen.js     # Дашборд
        ├── AddActivityScreen.js  # Форма добавления
        ├── HistoryScreen.js  # Статистика
        └── SettingsScreen.js # Настройки
```

### 3.2 Компонентная архитектура
Каждый экран — независимый функциональный компонент со своим состоянием:

```javascript
const ScreenName = ({ navigation }) => {
  // Локальное состояние
  const [data, setData] = useState(initialValue);
  
  // Побочные эффекты
  useEffect(() => {
    loadData();
  }, []);
  
  // Обработчики событий
  const handleAction = () => { ... };
  
  // Рендер
  return (
    <View>...</View>
  );
};
```

### 3.3 Навигационная структура
```
Stack Navigator
├── Home (Главная)
├── AddActivity (Добавить)
├── History (История)
└── Settings (Настройки)
```

## 4. Реализация ключевых функций

### 4.1 Управление состоянием (useState)
```javascript
// Хранение списка активностей
const [activities, setActivities] = useState([]);

// Хранение статистики
const [stats, setStats] = useState({
  total: 0,
  completed: 0,
  pending: 0,
});
```

### 4.2 Жизненный цикл компонента (useEffect)
```javascript
useEffect(() => {
  // Загрузка данных при монтировании
  loadActivities();
}, []); // Пустой массив зависимостей = выполнится один раз
```

### 4.3 Навигация между экранами
```javascript
// Переход на другой экран
navigation.navigate('AddActivity');

// Возврат назад
navigation.goBack();

// Доступ к параметрам
const { id } = route.params;
```

### 4.4 Glassmorphism эффект
Реализуется через комбинацию свойств:
```javascript
{
  backgroundColor: 'rgba(255, 255, 255, 0.7)',  // Полупрозрачность
  shadowColor: '#000',                          // Тень
  shadowOffset: { width: 0, height: 2 },
  shadowOpacity: 0.08,
  shadowRadius: 4,
  elevation: 3,                                  // Для Android
  borderWidth: 1,
  borderColor: 'rgba(255, 255, 255, 0.4)',      // Граница
}
```

## 5. Описание экранов

### 5.1 HomeScreen (Главная страница)
**Функции:**
- Отображение приветствия
- Карточка прогресса с процентом выполнения
- Список активностей на сегодня
- Интерактивные чекбоксы для отметки выполнения
- Плавающие кнопки навигации

**Технические особенности:**
- ScrollView для прокрутки контента
- FlatList для эффективного рендеринга списков
- TouchableOpacity для интерактивных элементов
- Dimensions.get('window') для адаптивности

### 5.2 AddActivityScreen (Добавление активности)
**Функции:**
- Форма с полями: название, категория, время, заметки
- Валидация обязательных полей
- Выбор категории через кликабельные чипы
- Alert для подтверждения сохранения

**Технические особенности:**
- TextInput для ввода текста
- Switch для переключателей
- Map-рендеринг для списка категорий
- Сброс формы после сохранения

### 5.3 HistoryScreen (История и статистика)
**Функции:**
- Карточки со статистикой (всего, выполнено, эффективность)
- Переключатель периода (день/неделя/месяц)
- Список истории с цветовыми индикаторами
- Определение топ-категории

**Технические особенности:**
- Вычисление статистики через reduce/filter
- Условная стилизация выполненных/невыполненных
- FlatList внутри ScrollView (scrollEnabled={false})

### 5.4 SettingsScreen (Настройки)
**Функции:**
- Переключатели настроек (уведомления, тема, звук)
- Выбор языка
- Кнопки экспорта/импорта/сброса данных
- Информация о приложении

**Технические особенности:**
- Switch компонент React Native
- Alert.alert для диалогов
- Секционированная верстка

## 7. Объяснение для защиты диплома

### Вопросы, которые может задать комиссия:

**1. Почему React Native, а не нативная разработка?**
- Кроссплатформенность (один код для iOS и Android)
- Быстрая разработка и итерации
- Горячая перезагрузка (Fast Refresh)
- Большое сообщество и экосистема

**2. Почему не использовали Redux/MobX?**
- Для учебного проекта достаточно локального состояния
- Упрощение архитектуры для демонстрации
- Легче объяснить каждый компонент
- Можно масштабировать при необходимости

**3. Как работает навигация?**
- Stack Navigator создает стек экранов
- При переходе новый экран кладется поверх предыдущего
- Кнопка "назад" убирает верхний экран из стека
- Параметры передаются через navigate('Screen', {params})

**4. Что такое glassmorphism?**
- Современный тренд UI-дизайна
- Эффект матового стекла через полупрозрачность
- Достигается rgba() + тени + границы
- Визуально выделяет элементы на фоне

**5. Как сохранить данные после закрытия приложения?**
- В текущей версии данные сбрасываются
- Для сохранения можно использовать:
  - AsyncStorage (простое ключ-значение хранилище)
  - SQLite (реляционная база данных)
  - Интеграция с бэкендом через API

**6. Как адаптировать под разные экраны?**
- Flexbox для гибкой верстки
- Dimensions.get() для получения размеров экрана
- Процентные ширины вместо фиксированных
- ScrollView для прокрутки длинного контента
